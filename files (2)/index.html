IT Security Audit & Compliance: Enterprise Controls Assessment

Botium Toys — Simulated Enterprise IT Audit

The Problem

Botium Toys, a growing international retailer, lacked a formalized security posture. This created significant exposure to data breaches and regulatory penalties under GDPR and PCI DSS, putting customer trust and business continuity at risk.

Methodology

I conducted a structured gap analysis across 14 critical IT security controls, including Least Privilege, Separation of Duties, and Encryption. Each control was mapped directly to SOC 2, PCI DSS, and GDPR requirements to identify compliance gaps and high-risk failure points.

Key Recommendation

The audit uncovered a critical lack of Disaster Recovery planning and weak encryption protocols. I developed a remediation roadmap prioritizing MFA implementation, encrypted data handling, and automated backups to improve resilience and align the organization with SOC 2 controls.

Incident Response: Mitigating a DDoS Attack via NIST Framework

Applied the NIST Cybersecurity Framework (CSF) to investigate, contain, and recover from a simulated distributed denial-of-service attack.

The Incident

At 09:00, all network services became unresponsive. Traffic analysis revealed a malicious ICMP flood attack saturating network resources and causing a complete service outage across critical systems.

Response Framework (NIST CSF)

Identify

Determined that ICMP flood traffic was the root cause of the outage, impacting 100% of available network resources.

Protect / Detect

Implemented firewall rate limiting and source IP verification to mitigate spoofed ICMP traffic and prevent further saturation.

Recover

Prioritized restoration of critical services before reintroducing non-critical systems once attack traffic subsided.

Outcome & Key Takeaways

The structured NIST CSF approach enabled rapid triage, containment, and recovery. The incident reinforced the importance of proactive traffic filtering, network monitoring, and layered defensive controls.

Automated Cloud Data Pipeline & Retail Intelligence

Production-grade ETL pipeline leveraging AWS infrastructure to automate retail data flow from S3 to RDS for real-time business intelligence

Executive Summary

I developed a production-grade ETL (Extract, Transform, Load) pipeline that automates the flow of retail data from a cloud data lake into a high-performance relational database. By normalizing flat-file data into a Star Schema, I optimized the data for business intelligence, enabling real-time analysis of holiday sales trends and macroeconomic impacts.

Architecture & The Data Journey

The project mimics a modern corporate data stack, moving data through three distinct layers:

Extraction (Raw Layer)

Utilized AWS S3 as a scalable data lake to store raw CSV sales data, ensuring the data is decoupled from local hardware and accessible globally.

Transformation (Processing Layer)

Engineered a Python ETL script that programmatically cleans the data and reorganizes it into a 3rd Normal Form (3NF) relational structure.

Loading (Warehouse Layer)

Automated the data load into an AWS RDS (PostgreSQL) instance using SQLAlchemy, creating a centralized "Source of Truth" for analysts.

Database Engineering (SQL)

To ensure data integrity and query performance, I transitioned the data from a single "flat" file into three related tables. This reduces data redundancy and allows for more complex, efficient reporting:

stores: Dimension table containing unique location metadata
economic_indicators: Tracks CPI, Fuel Prices, and Unemployment over time
sales_transactions: Fact table capturing weekly revenue per store

Implemented Primary and Foreign Key relationships within the RDS environment to maintain referential integrity.

Data Visualization & Insights (Power BI)

I connected the data layer to Power BI to translate rows and columns into actionable business insights:

Top-Tier Performers: Top 10% of stores contribute disproportionately to total revenue, suggesting inventory optimization opportunities

Technical Challenges & Solutions

Challenge: Connecting local BI tools to cloud databases often involves SSL certificate hurdles.

Solution: I navigated AWS Security Group (firewall) configurations and implemented an ODBC connection string with specific SSL parameters to ensure a secure, successful data handshake between Power BI and the AWS Cloud.
